6- What is the difference between a VM and a Container? 

= A VM has its own full OS, which makes it heavy and slow to start. A container shares the host OS kernel, so it's lightweight and starts in seconds. Docker containers are the most common example.
---

7- What is CI/CD? Explain it simply. 

= CI (Continuous Integration) means automatically testing and building code every time a developer pushes code. CD (Continuous Delivery/Deployment) means automatically deploying that tested code to servers. Together they remove manual steps and speed up releases.
---

8- What is Docker and what problem does it solve? 

= Docker is a tool that packages your application and all its dependencies into a container so it runs the same way everywhere — on your laptop, on a server, or in the cloud. It solves the classic "it works on my machine" problem.
---

9- What is Kubernetes and why do we need it if we already have Docker?

= Docker runs containers. Kubernetes manages hundreds of containers across many servers. It handles scaling, self-healing (restarting failed containers), load balancing, and rolling updates automatically.

 10- What is the difference between Git merge and Git rebase?

 = Both integrate changes from one branch into another. Merge keeps the full history and creates a merge commit. Rebase rewrites history to look like a straight line — cleaner, but don't use it on shared branches.
